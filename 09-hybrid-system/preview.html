Hybrid System: Agent + RAG + Tools

This document describes a hybrid AI system that combines:

  • Agent / Router for deciding how to handle a question
  • Tools for structured operations like ICD lookup and reimbursement lookup
  • RAG for answering knowledge questions from documents
  • LLM synthesis for producing a final answer from gathered evidence
Core idea: not every question should be handled the same way. Some questions are better served by structured tools. Others need retrieval from documents. Some need both.

What This System Should Handle

  • What is the ICD code for chest pain?
  • What is the reimbursement for R07.9?
  • What is the reimbursement for chest pain?
  • What are EBM codes used for?
  • Can smoking make asthma worse?

The system should answer these questions by choosing the right combination of:

  • tool calls for structured lookups
  • RAG for document-based knowledge
  • LLM reasoning to combine results into a natural answer

High-Level Architecture

User Question
   ↓
Agent / Router
   ↓
Decide:
- Use tool?
- Use RAG?
- Use both?
   ↓
Gather evidence
   ↓
Generate final answer

This is a controlled hybrid design. It is intentionally simpler and clearer than a fully autonomous agent stack.

Component Responsibilities

1. Tools

Tools handle structured business logic that should be deterministic and trusted.

  • lookup_icd_code(term) maps a symptom or diagnosis term to an ICD code
  • estimate_reimbursement(icd_code) maps an ICD code to a reimbursement estimate

2. RAG Engine

RAG handles knowledge-style questions by retrieving relevant information from documents in the data/ folder and using that as context for an LLM.

3. Router

The router inspects the user’s question and decides whether the system should:

  • use a structured tool
  • use RAG
  • use both tools and RAG in sequence

4. Synthesizer

The synthesizer is the final LLM call that turns gathered evidence into a clean answer.

Tools Code

tools/icd.py

def lookup_icd_code(term: str) -> dict:
    codes = {
        "chest pain": {"code": "R07.9", "description": "Chest pain, unspecified"},
        "hypertension": {"code": "I10", "description": "Essential (primary) hypertension"},
        "asthma": {"code": "J45.909", "description": "Unspecified asthma, uncomplicated"},
    }

    key = term.strip().lower()
    if key in codes:
        return {"found": True, "term": key, **codes[key]}
    return {"found": False, "message": f"No ICD code found for '{term}'"}

tools/reimbursement.py

def estimate_reimbursement(icd_code: str) -> dict:
    estimates = {
        "R07.9": {"estimated_amount": 125, "currency": "USD"},
        "I10": {"estimated_amount": 95, "currency": "USD"},
        "J45.909": {"estimated_amount": 140, "currency": "USD"},
    }

    code = icd_code.strip().upper()
    if code in estimates:
        return {"found": True, "icd_code": code, **estimates[code]}
    return {"found": False, "message": f"No reimbursement estimate found for '{code}'"}

RAG Engine Code

rag/engine.py

from llama_index.core import VectorStoreIndex, SimpleDirectoryReader, PromptTemplate

def build_query_engine():
    documents = SimpleDirectoryReader("data").load_data()
    index = VectorStoreIndex.from_documents(documents)

    qa_template = PromptTemplate(
        "Use the context to answer the question. "
        "You may combine multiple facts. "
        "If insufficient, say so.\\n\\n"
        "Context:\\n{context_str}\\n\\n"
        "Question: {query_str}\\n"
        "Answer:"
    )

    return index.as_query_engine(text_qa_template=qa_template)
What this does: loads documents, builds a vector index, and returns a query engine that can retrieve relevant chunks and answer questions from them.

Router Code

agent/router.py

def classify_query(query: str) -> str:
    q = query.lower()

    if "icd" in q and "reimbursement" not in q:
        return "icd_lookup"

    if "reimbursement" in q and "icd" in q:
        return "reimbursement_by_icd"

    if "reimbursement" in q:
        return "reimbursement_by_symptom"

    return "rag"

This is intentionally simple and deterministic. It is not meant to be “smart” yet. It is meant to be clear and debuggable.

Main Orchestration Code

main.py

import os
from dotenv import load_dotenv
from openai import OpenAI

from tools.icd import lookup_icd_code
from tools.reimbursement import estimate_reimbursement
from rag.engine import build_query_engine
from agent.router import classify_query

load_dotenv()
client = OpenAI(api_key=os.getenv("OPENAI_API_KEY"))

query_engine = build_query_engine()

def synthesize_answer(query: str, evidence: str) -> str:
    response = client.responses.create(
        model="gpt-4.1-mini",
        input=[
            {
                "role": "system",
                "content": (
                    "You are a healthcare AI assistant. "
                    "Use the provided evidence to answer clearly and concisely. "
                    "If evidence is incomplete, say so."
                )
            },
            {
                "role": "user",
                "content": f"Question: {query}\\n\\nEvidence:\\n{evidence}"
            }
        ]
    )
    return response.output_text

query = input("Ask: ")
route = classify_query(query)

print(f"\\nRoute selected: {route}")

if route == "icd_lookup":
    term = query.lower().replace("what is the icd code for", "").strip(" ?")
    result = lookup_icd_code(term)
    evidence = str(result)
    answer = synthesize_answer(query, evidence)

elif route == "reimbursement_by_icd":
    words = query.replace("?", "").split()
    icd_code = None
    for w in words:
        if "." in w or w.upper().startswith(("R", "I", "J")):
            icd_code = w
            break

    if icd_code:
        result = estimate_reimbursement(icd_code)
    else:
        result = {"found": False, "message": "No ICD code identified in query."}

    evidence = str(result)
    answer = synthesize_answer(query, evidence)

elif route == "reimbursement_by_symptom":
    term = query.lower().replace("what is the reimbursement for", "").strip(" ?")
    icd_result = lookup_icd_code(term)

    if icd_result.get("found"):
        reimb_result = estimate_reimbursement(icd_result["code"])
        evidence = f"ICD lookup result: {icd_result}\\nReimbursement result: {reimb_result}"
    else:
        evidence = f"ICD lookup result: {icd_result}"

    answer = synthesize_answer(query, evidence)

else:
    rag_response = query_engine.query(query)
    evidence = str(rag_response)
    answer = synthesize_answer(query, evidence)

print("\\nAnswer:\\n")
print(answer)

How the Main Flow Works

The system follows this sequence:

  1. Ask the user for a question
  2. Classify the question using the router
  3. Choose the correct path:
    • ICD lookup
    • reimbursement by ICD
    • reimbursement by symptom
    • RAG-based knowledge answer
  4. Gather evidence from tools and/or RAG
  5. Pass the evidence to the LLM for final answer generation

Why This Architecture Is Good

Clear separation of concerns:
  • Router decides the path
  • Tools perform structured deterministic work
  • RAG handles document-based knowledge
  • LLM turns evidence into a user-facing answer

This is a strong architecture because it avoids mixing:

  • routing logic
  • business logic
  • knowledge retrieval
  • response writing

  • Route: rag
  • RAG retrieves relevant text from documents
  • LLM answers from retrieved content

5. Can smoking make asthma worse?

  • Route: rag
  • RAG retrieves supporting facts from documents
  • LLM synthesizes the answer from those facts

Why Not Use a Fully Autonomous Agent Yet?

At this stage, the goal is architecture clarity, not maximum autonomy.

A fully autonomous system that mixes:

  • tool calling
  • RAG
  • multi-step looping
  • framework abstractions

can become hard to debug very quickly.

This controlled hybrid design is better because you can clearly see:

  • which route was selected
  • which evidence was gathered
  • how the final answer was produced
